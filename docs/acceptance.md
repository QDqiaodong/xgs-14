# 验收说明

本文档对照 `mvp.md` 与 `mvp.json` 的验收标准，给出实现结果与证据。

## 自动化验证入口

```bash
pnpm test
```

`pnpm test` 覆盖：

- 认证与鉴权（401/403）
- 顾客下单与金额计算
- 商家菜品查询/新增/删除/上下架
- 顾客取消自己的 NEW 订单
- 商家订单状态更新并对顾客可见
- 下架菜品下单失败且不落库
- 购物车本地持久化与金额计算逻辑（`scripts/cart.test.mjs`）
- 全页面分支与接口边界由 Playwright 覆盖（见 `docs/e2e-coverage-matrix.md`）

## AC 对照

### AC-CORE-001

- 结论：通过
- 证据：`scripts/e2e.mjs` 中顾客创建订单后断言 `status === NEW`，并校验顾客订单列表新增。

### AC-CORE-002

- 结论：通过
- 证据：商家创建可售菜品后，顾客调用 `GET /api/dishes` 能看到该菜品。

### AC-EDGE-001

- 结论：通过
- 证据：商家下架菜品后，顾客提交包含该菜品订单返回 `409/40900`，并断言订单总数不变。

### AC-ERROR-001

- 结论：通过
- 证据：未登录访问 `GET /api/orders` 返回 `401/40100`；前端 `api.js` 在受保护页面收到 401 自动跳转登录页。

### AC-USABILITY-001

- 结论：通过
- 证据：`web/assets/js/cart.js` 每次增减/移除即时重算总价；`scripts/cart.test.mjs` 验证 localStorage 持久化恢复。

### AC-CORE-003

- 结论：通过
- 证据：商家将订单状态更新为 `CONFIRMED` 后，顾客查询同订单状态已变更。

### AC-CORE-004

- 结论：通过
- 证据：顾客订单页新增“取消订单”按钮，顾客可将自己的 `NEW` 订单更新为 `CANCELLED`。

### AC-EDGE-002

- 结论：通过
- 证据：商家菜品页支持关键字/状态查询；未被历史订单引用的菜品可删除，已被引用的菜品删除返回 `409/40900` 并提示改为下架。

### AC-ERROR-002

- 结论：通过
- 证据：顾客调用 `POST /api/dishes` 返回 403；商家调用 `POST /api/orders` 返回 403。

## 手工验收建议

1. 使用商家账号登录，按关键字/状态查询菜品，新增一条未被引用的菜品后执行删除。
2. 注册顾客账号，点餐并下单。
3. 顾客刷新页面确认购物车恢复，并在“我的订单”中取消一笔 `NEW` 订单。
4. 商家更新其他订单状态，顾客刷新订单页观察状态变化。
5. 顾客尝试访问商家接口，验证 403。
